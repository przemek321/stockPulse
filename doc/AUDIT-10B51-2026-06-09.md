# Audyt 10b5-1 backfill — 09.06.2026 (Pakiet 1 fix #0)

## Kontekst

Odkrycie P0 z badania 09.06 ([PLAN-EDGE-IMPROVEMENTS-2026-06-09.md](PLAN-EDGE-IMPROVEMENTS-2026-06-09.md) §1.1):
parser produkcyjny (`form4-parser.ts`) i backtestowy (`edgar_fetcher.py`) czytały
per-transaction tag `Rule10b5-1Transaction`, który **nie występuje w realnych filingach
EDGAR**. Prawdziwy znacznik planu to doc-level checkbox `<aff10b5One>` (obowiązkowy od
amendmentu SEC, kwiecień 2023). Efekt: **0/3394** wierszy `insider_trades` miało
`is10b51Plan=true` — filtr "discretionary only" w Form4Pipeline był no-opem.

Fix wdrożony 09.06.2026: oba parsery czytają doc-level `<aff10b5One>` (zbiór wartości
`'1'/'true'/'y'`, case-insensitive, spójny TS↔Python) z zachowanym per-transaction
fallbackiem. Zweryfikowany adwersarialnie (3 soczewki, w tym empiryczny test
fast-xml-parser + 7 realnych filingów z SEC: casing, brak namespace, pojedyncze
wystąpienie elementu — wszystko potwierdzone).

## Metoda audytu

`scripts/audit/audit_10b51_backfill.py` — read-only: dla każdego z **1248 unikalnych
filingów** w `insider_trades` pobrany surowy XML z SEC EDGAR (rate limit ~4 req/s),
sprawdzony `<aff10b5One>` + per-transaction fallback. 15 filingów bez `documentUrl`
odzyskanych przez mapę ticker→CIK (`company_tickers.json`). 1 wiersz odrzucony
(accession `mspr` — artefakt data quality, ABBV). Wyniki per filing:
`scripts/audit/aff_flags.csv`.

## Wnioski

1. **BUY edge jest czysty z konstrukcji** — 0/18 BUY pochodziło z planów. Zmierzony
   edge (V5 C-suite BUY d=+0.92, APLS d=+0.75) nie wymaga korekty o plany; "all BUY"
   ≡ "discretionary BUY" w naszych danych. Obawa o mieszane filingi (discretionary BUY
   co-filed z planowym SELL, wycinany przez doc-level flagę) skwantyfikowana: **0 przypadków**.
2. **80% SELL to plany** — wszystkie wnioski systemowe o SELL (Director SELL anty-sygnał,
   C-suite SELL zero edge H2, suppression gates) były mierzone na populacji 4:1
   zdominowanej przez plany. Produkcyjna decyzja (tłumienie SELL) pozostaje bezpieczna —
   nawet zanieczyszczona populacja nie miała edge'u — ale **liczby d dla SELL nie opisują
   discretionary SELL**. Po fixie do GPT trafi ~5× mniej SELL (tylko 20% discretionary).
3. **GILD O'Day 29.04 (trigger S19-FIX-07) potwierdzony jako plan** (`aff10b5One=1`) —
   system alertował planową sprzedaż jako discretionary CRITICAL.
4. **Koszt GPT spadnie** — plany są skipowane PRZED wywołaniem GPT (krok 2 decision tree),
   a stanowiły ~22% wszystkich transakcji (752/3394).

## Backfill DB — ✅ WYKONANY 10.06.2026 (zgoda Przemka)

Przebieg (szczegóły w logu na końcu `scripts/db/backfill-10b51-2026-06-09.sql`):
1. Pierwsze wykonanie skryptu: `UPDATE 488` — **mniej niż oczekiwane 752**. Diagnoza:
   warunek `is10b51Plan = false` pomijał **1512 wierszy z NULL** (stare kolekcje
   sprzed defaulta kolumny; NULL ≠ false w SQL).
2. Dograne: plan z `IS DISTINCT FROM true` (+264 = komplet 752) + **normalizacja**
   NULL → false dla wierszy z audytowanych filingów discretionary (1246 wierszy).

**Stan końcowy** (zgodny z audytem + nowe kolekcje post-deploy): plan **758**
(552 SELL / 143 EXERCISE / 33 TAX / 22 GRANT / 8 OTHER / **0 BUY**), NULL zostały
tylko 2 artefakty `Aggregate MSPR` (mspr_ABBV_*, nie-SEC — celowo nietknięte).
Od teraz 30-dniowa historia w promptach GPT pokazuje poprawne flagi z anotacją
`[10b5-1 plan]`, a retro-analizy plan-vs-discretionary na DB są możliwe.

Odwracalność: `UPDATE insider_trades SET "is10b51Plan"=false WHERE "is10b51Plan"=true`
(plus przywrócenie NULL nieodtwarzalne, ale NULL≡false semantycznie w całym kodzie —
`t.is10b51Plan ?? false`).

## Decyzja o re-run V5

Plan przewidywał re-run "sanity: czy BUY edge przeżywa poprawne flagowanie" — **BUY
przeżywa z konstrukcji** (0 planów), więc pilność spada. Re-run pozostaje wartościowy dla:
- **H3 plan-vs-discretionary**: w V5 grupa `10b51_plan` miała **n=0** (ten sam bug
  parsera w `edgar_fetcher.py`) — hipoteza uzasadniająca produkcyjny skip nigdy nie
  porównała realnych grup. Po fixie porównanie wreszcie możliwe.
- **Discretionary SELL**: czy czyste 20% SELL ma edge, którego nie widać w rozwodnionej
  populacji (mało prawdopodobne wg literatury — Lakonishok & Lee: SELL bez wartości
  predykcyjnej — ale teraz mierzalne).

⚠️ Re-run wymaga **force re-fetch**: `scripts/backtest/data/form4_transactions.csv`
(40 874 wierszy, 0 flagowanych) jest cache'owany — fix parsera nie działa bez usunięcia
cache (potwierdzono w weryfikacji adwersarialnej). Koszt: godziny fetchowania SEC.
Rekomendacja: razem z przyszłym backtestem pivotu discovery (Pakiet 2), nie jako osobny run.
